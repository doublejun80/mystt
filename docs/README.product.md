# mystt product notes

## Purpose

mystt is a meeting recording workspace built around one priority: keeping the
original audio alive long enough for reliable upload, transcription, and review.

The product should help a user start recording quickly first, then clean up
titles, review transcripts, generate notes, download artifacts, and share output
after the recording is safely saved.

## Product priorities

1. Preserve original audio before polish.
2. Make upload and transcription failures recoverable.
3. Keep the mobile recording screen focused on the recording controls.
4. Keep explanatory copy out of the first viewport when it competes with
   recording actions.
5. Put operational details, provider behavior, and tuning options in settings
   or documentation instead of crowding the capture flow.

## UI principle

The first mobile viewport should answer one question: "Can I start or stop the
recording right now?"

Secondary actions such as refresh, logout, settings, provider status, and mode
details should stay compact or move out of the way.
