## Shared Capture Checklist

- 세션 ID와 run ID를 먼저 적는다.
- 자동 검증 결과와 수동 증거를 분리해서 적는다.
- 상태 파일은 원본 경로까지 남긴다.
- 모바일은 `runtime-state.json`과 `recordings/<session-id>/session.json`을 같이 남긴다.
- 모바일 세션은 `checksumMd5`와 `uploadQueuedAt`를 확인한다. `checksumMd5`는 모바일 로컬 저장소에서 원본 파일 무결성을 가리키는 신호로 적고, 업로드 검증이 포함된 run에서는 API / Soniox 쪽 `sha256`도 같이 적는다.
- `backgroundTransitionCount`, `transportState`, `phase`, `lastKnownAppState`, `selectedInput`를 기록한다.
- desktop은 `appDataDir`, `recorderRoot`, `recordingsRoot`, `sessions.json`, `runtime-state.json`을 기록한다.
- desktop은 `saved_session_count`와 `recent_sessions`가 기대값인지 확인한다.
- keep-awake 상태와 target URL을 함께 적는다.
- Soniox live는 업로드와 cleanup을 둘 다 확인한다.
- `source_audio.soniox_uploaded`와 `transcription.metadata.updated`, `transcription.cleanup.updated`를 같이 적는다.
- `cleanupStatus`, `cleanupTargets`, `cleanupLastError`, `session.process.enqueued`를 빠뜨리지 않는다.
- 실패하면 rollback 또는 재시도 기준을 메모한다.

## How To Use

1. `evidence-ledger.md`의 해당 row를 복사해 새 run을 만든다.
1. `_template.md`를 붙여 넣고 값만 채운다.
1. lane README의 체크리스트와 비교해 빠진 증거가 없는지 확인한다.
1. 수동 증거는 스크린샷, 로그 excerpt, 상태 파일 경로, webhook / API 응답 중 최소 1개 이상을 포함한다.
